# AGENTS メモ

更新日: 2026-07-12  
対象: Open WebUI と連携する業務ポータルの新規構築

## プロジェクトの目的

現行 PHP / MySQL システムをそのまま移植せず、案件単位で資料・構造化データ・対話・成果物を安全に扱える業務ポータルへ再設計する。実施計画は [業務ポータル開発計画](docs/frontend_delivery_plan.md) を正本とする。

- Open WebUI: 改造しない。認証済みチャット、モデル接続、標準ファイル管理、Knowledge/RAG、会話UI
- 独自フロントエンド: 業務ポータルとして案件、資料/CSV、成果物、進捗、承認を扱う。チャットUIは複製しない
- 独自サービス: 案件ACL、CSV台帳・集計、帳票、外部データ取込、非同期ジョブ、監査
- 連携方式: バージョン付き OpenAPI Tool を第一候補とし、Open WebUI Function は薄い接続・表示補助に限る

## 初期実行計画

### Phase 0: 契約とプロジェクト基盤

1. 業務ポータル、業務API、Open WebUIの公開境界を定義する。
2. 開発・検証環境、秘密情報の管理方法、ログの保存先を決める。
3. Open WebUI 本体を改変せずに利用し、対象バージョンを固定する。

完了条件: リポジトリ構成、起動前提、作業規約、最初の検証項目が文書化されている。

### Phase 1: P0業務ポータルと境界を検証する最小PoC

1. 業務ポータルで案件一覧・案件詳細・メンバー・資料/成果物/ジョブ一覧を実装する。
2. FastAPI と業務DBで、案件一覧・案件権限確認・CSV安全集計の最小APIを実装する。
3. Open WebUIで予定モデル、PDF抽出、Knowledgeの日本語検索を評価し、OpenAPI Toolを接続する。

完了条件: 案件を選択した利用者が、ポータルで資料・成果物・進捗を確認し、権限のあるCSV集計をチャットから実行できる。根拠・監査イベント・拒否理由を確認できる。

### Phase 2: 成果物と非同期処理

1. 資料・CSV・帳票のメタデータと実体の保存先を分離する。
2. OCR、解析、PDF生成をキューとワーカーで実行し、ジョブID・状態・失敗理由・再試行を記録する。
3. 構造化した `ReportDraft` を利用者確認後にPDFへ出力する。

完了条件: 長時間処理を会話リクエスト内で完走させず、ポータルで成果物と進捗を確認できる。

### Phase 3: 更新互換性と拡張

1. Open WebUI更新前のステージング回帰を自動化し、ポータル/APIとは別々にデプロイする。
2. FAQ、CSV統合・AI分類、外部DB取込、図解を、利用実績と安全性を確認しながら追加する。
3. 多段推論、LLM judge、横断調査はPoC評価後に採否を判断する。

## 実装・運用の規則

1. Plan → Do → Check/Debug → Act の順で進める。変更前に目的、対象、完了条件、更新する文書を明確にする。
2. Open WebUI の内部DB、非公開API、DOM/CSS、本体ソース変更を業務機能の契約にしない。
3. 業務APIは、利用者・案件・操作権限をすべて再検証する。クライアント入力の `project_id`、SQL、権限情報を信頼しない。
4. Tool API はバージョン付きOpenAPIスキーマで管理する。重い処理は即時に `job_id` を返す非同期契約にする。
5. SQLは読み取り専用アカウント、許可ビュー/テーブル、allowlist、行数・実行時間制限を必須にする。
6. 秘密情報、トークン、個人情報、DBダンプをリポジトリ・ログ・ドキュメントに含めない。
7. 仕様決定は `docs/`、未完了作業は `TODO.md`、障害の再発防止は `TROUBLESHOOTING.md` に記録する。
8. Open WebUI更新前にステージングで、ログイン、Knowledge、Tool、会話ストリームを回帰確認する。
9. Open WebUI本体はforkせず、Docker/Pythonの同一固定バージョンを検証して更新する。詳細は `docs/runtime_management.md` に従う。
10. 業務ポータル/APIの正規の開発・テスト・デバッグ環境はDocker Composeとする。利用者向けWindows配布ではDockerを要求せず、Open WebUIのPython環境とは別の専用venv/実行ファイルで業務アプリを起動する。詳細は `docs/docker_development_environment_plan.md` と `docs/windows_distribution_plan.md` に従う。
